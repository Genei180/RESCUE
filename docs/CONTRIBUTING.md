# Contributing Guide

👋 Thanks for considering contributing!

# for the Current State of the Project

Please read [Way Forward](./../README.md#Way%20Forward) first.

# When the Architectural Design Document and Kickoff is Finished we can proceed with the following Contributing Guidelines:

## Ways to Contribute
- Suggest big changes via RFCs
- Improve documentation
- Submit code via Pull Requests
- Report bugs via Issues


## Workflow
1. Fork the repo and create a feature branch.
2. For small changes, open an Issue or PR directly.
3. All code changes must go through Pull Requests.
4. For big changes, start with an RFC (see `Issues -> RFC Template`).
5. Decisions are documented as ADRs.

## Development
- Branch naming: `feature/*`, `fix/*`
- Run tests with: `make test` (or define your test command)
